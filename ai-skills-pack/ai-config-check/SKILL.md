# 人工智能规范体检

`ai-config-check` 用来检查一个项目的 AI 协作规范是否完整、可执行、可持续。

它承接 `ai-bootstrap` 的七层模型：

1. 宪法
2. 规范
3. 上下文
4. 工具
5. 护栏
6. 验证
7. 技能

输出重点不是分数，而是每一层的状态、证据、缺口和优化建议。

---

## 工作流程

```text
阶段 1：采集和评分
阶段 2：七层诊断报告
阶段 3：可选离线 HTML 仪表盘
```

过程记录默认写临时目录，不写入项目。只有用户明确要求保存报告或仪表盘时，才把文件写入项目目录。

---

## 阶段 1：采集和评分

运行脚本：

```bash
python3 "${AI_SKILL_ROOT}/scripts/run_checks.py" "$(pwd)" --config-dir <CONFIG_DIR>
```

必须先判断目标工具，再显式传入 `CONFIG_DIR`，例如 `.codex`、`.cursor` 或 `.claude`。不要让脚本替用户猜配置目录。

脚本输出 JSON，核心字段：

- `layers`：七层诊断结果。
- `summary`：各状态数量。
- `total_score`：0-100 辅助分。
- `priority_actions`：最多 5 条优先优化动作。

分数规则：

- 每层 0-100 分。
- 七层等权汇总。
- 分数只用于帮助排序，不代表 KPI。

状态规则：

| 状态 | 含义 |
|------|------|
| 已具备 | 当前内容能指导 AI 稳定执行 |
| 缺失 | 没有发现对应入口 |
| 需补齐 | 有入口，但缺少关键事实、命令、边界或验证 |
| 冲突 | 多处事实不一致，需要先确认事实源 |

---

## 阶段 2：七层诊断报告

报告必须按以下结构输出：

```markdown
# AI 规范体检报告

- 项目：<项目名>
- 总分：XX / 100
- 总体状态：<一句话>

## 七层状态

| 层级 | 状态 | 得分 | 重点 |
|------|------|------|------|
| 宪法 | 需补齐 | 60 | 有入口但缺交付标准 |
| ... | ... | ... | ... |

## 详细诊断

### 宪法：需补齐（60/100）

证据：
- <发现了什么>

问题：
- <缺口或冲突>

优化建议：
- <可执行建议>

## 优先优化动作

1. <最重要的动作>
2. ...
```

每层必须告诉用户怎么优化。不要只输出分数。

---

## 阶段 3：离线 HTML 仪表盘（可选）

用户需要可视化时再生成。

```bash
python3 "${AI_SKILL_ROOT}/scripts/run_checks.py" "$(pwd)" --config-dir <CONFIG_DIR> \
  --output /tmp/ai-config-check-results.json

python3 "${AI_SKILL_ROOT}/scripts/render_dashboard.py" \
  --results /tmp/ai-config-check-results.json \
  --output /tmp/ai-config-check-dashboard.html
```

HTML 仪表盘必须是自包含静态页面，不依赖 CDN、外部 JavaScript 或网络资源。生成项目内 HTML 前必须确认用户需要保存该文件。否则输出到 `/tmp`。

---

## 注意事项

- 分数不是 KPI，不做团队排名。
- 对已有宪法项目，重点检查是否缺层、冲突、过期、不可执行。
- 发现冲突时先停下来指出冲突来源，不要直接替用户选事实源。
- 护栏必须是 AI 外部自动机制；纯文本规则不是护栏。
- 过程记录默认不写入项目。
